# 🛠️ 핵심 기술 스택 요약 (LLM, Docker, Kafka, Redis)

금융 플랫폼 엔지니어 면접에서 기술의 본질과 활용 사례를 간결하게 답변할 수 있도록 핵심 위주로 정리했습니다.

---

## 🤖 1. LLM (Large Language Model)
*   **한 줄 정의**: 방대한 텍스트 데이터를 학습하여 인간처럼 언어를 이해하고 생성하는 거대 인공지능 모델.
*   **핵심 원리**: 다음 단어가 무엇일지 확률적으로 예측하는 'Transformer' 아키텍처 기반.
*   **금융/퀀트 활용**:
    *   **감성 분석(Sentiment Analysis)**: 뉴스, 리포트, 공시 자료를 읽고 시장의 긍정/부정 분위기를 데이터화하여 투자 지표로 활용.
    *   **코드 생성 및 자동화**: 복잡한 퀀트 수식이나 백테스팅 코드를 초안 작성하여 개발 생산성 향상.
    *   **비정형 데이터 구조화**: PDF나 텍스트로 된 재무제표 데이터를 정형 데이터(DB)로 변환.

---

## 🐳 2. Docker (도커)
*   **한 줄 정의**: 애플리케이션과 실행에 필요한 모든 환경(라이브러리, 설정 등)을 하나로 묶어 어디서나 동일하게 실행하게 해주는 컨테이너 기술.
*   **핵심 가치**: "내 컴퓨터에선 되는데 서버에선 안 돼요"라는 환경 문제를 해결 (표준화).
*   **금융/퀀트 활용**:
    *   **연산 무결성 보장**: 수백 대의 백테스팅 서버가 동일한 버전의 Pandas/NumPy를 사용하도록 강제하여 계산 결과의 일관성 유지.
    *   **리소스 격리**: 무거운 연산 작업이 다른 서비스에 영향을 주지 않도록 CPU/메모리 자원을 제한.
    *   **빠른 확장**: 쿠버네티스(K8s)와 결합하여 연산량이 많아질 때 순식간에 워커 노드를 수백 개로 늘림.

---

## 📨 3. Kafka (카프카)
*   **한 줄 정의**: 대용량의 데이터를 실시간으로 안정적으로 전달하는 고성능 분산 메시지 브로커.
*   **핵심 원리**: 발행-구독(Pub-Sub) 모델과 디스크 로그 저장 방식을 통해 데이터 유실 없이 대량 처리가 가능.
*   **금융/퀀트 활용**:
    *   **실시간 시세 파이프라인**: 거래소에서 초당 수만 건씩 쏟아지는 호가/체결 데이터를 안정적으로 받아 여러 시스템(주문, 리스크 관리, DB 저장)에 배달.
    *   **시스템 간 비동기 결합**: 주문 시스템과 정산 시스템 사이의 직접적인 연결을 끊어(Decoupling), 한쪽이 죽어도 데이터가 유실되지 않게 함.

---

## ⚡ 4. Redis (레디스)
*   **한 줄 정의**: 메모리 상에 데이터를 저장하여 읽기/쓰기 속도가 압도적으로 빠른 인메모리(In-memory) 데이터 저장소.
*   **핵심 원리**: RAM에 데이터를 저장하여 디스크 기반 DB보다 수백 배 빠른 밀리초(ms) 단위 응답 제공.
*   **금융/퀀트 활용**:
    *   **실시간 팩터 캐싱**: 실시간 PBR, PER 등 계산량이 많은 지표를 미리 계산해 Redis에 넣어두고, 주문 엔진이 즉시 가져갈 수 있게 함 (초저지연 서빙).
    *   **분산 락(Distributed Lock)**: 여러 서버가 동시에 같은 계좌의 주문을 낼 때 중복 처리가 되지 않도록 제어.
    *   **세션 관리**: 글로벌 지사의 사용자들이 빠른 속도로 로그인 상태를 유지하도록 관리.
